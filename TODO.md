# TODO лист для реализации системы мониторинга датчиков

## Анализ текущего состояния проекта

У вас уже есть хорошая базовая структура:

1. **Collector** - собирает данные с датчиков и имитаторов (базовая структура)
2. **Core** - API сервис с FastAPI для обмена данными между компонентами
3. **Sensors** - модуль для классов сенсоров с абстрактным базовым классом `SensorValue`
4. **Predictor** - сервис для предсказаний (простая структура)
5. **Alarm** - два типа алертов (console и telegram)

Core API уже имеет базовую реализацию с эндпоинтами для:
- получения данных с сенсоров
- управления алертами
- запросов предсказаний
- проверки статуса системы
